# AI 時代生存指數 Image Bible

## 1. 整體視覺風格

風格方向：

- tarot-card inspired oracle illustration
- symbolic cyber fantasy
- mystical and spiritual
- AI technology mysticism
- neural network constellations
- holographic circuitry
- warm futuristic
- LINE friendly
- high contrast
- cinematic lighting
- not too dark
- clear archetype silhouette or central symbol
- suitable for 390px mobile screen

畫面要像神諭牌、塔羅牌與 LINE LIFF 小遊戲的混合，而不是企業簡報、科技報告或具體漫畫人物。每張圖應優先表現「原型能量」與「靈性象徵」，但必須混合 AI 科技元素：神經網路星座、資料光路、全息幾何、演算法符文、電路感光線。比例建議是 55% 神秘 / 45% AI 科技，避免變成純玄學或純科幻。

共通構圖規則：

- 優先使用塔羅牌式構圖：邊框、中心符號、幾何儀式感、星圖、光門、能量環。
- 每張圖都要有可辨識的 AI 成分：資料節點、神經網路、電路光線、全息介面或 AI 核心。
- 可以使用抽象斗篷剪影、動物靈或圖騰，但避免具體臉孔與漫畫吉祥物。
- 背景保留簡單的未來城市、符號陣列、光軌、星圖或能量環。
- 畫面中央 65% 區域留給原型符號、圖騰或情境主體。
- 上下 15% 保留 UI 疊字安全區。
- 避免過多小字、複雜圖表、真實品牌標誌、普通人像、寫實肖像。

### 1.1 AI 科技家徽 / 靈獸圖騰規則

九大原型的核心視覺不是角色插畫，而是「科技版家徽」。概念接近日本家徽的精神：極度符號化、對稱、可辨識、可縮小、像一個家族或流派的身份印記；但材料語言改成 AI 時代的神經網路、全息電路、資料光路與演算法幾何。

動物不是可愛吉祥物，也不是寫實動物肖像，而是「AI 靈獸圖騰」。動物只用輪廓、羽翼、角、眼、爪、流線、星座連線或能量場暗示，並與中央 AI 核心、神經網路、全息電路融合。

科技家徽表現原則：

- 優先做成一枚「徽記」，不是一張完整場景插畫。
- 圖形必須極簡、乾淨、利落；一眼看起來要像單一幾何靈獸紋章。
- 構圖接近家徽：正圓、菱形、三角、環形、放射、左右對稱或旋轉對稱。
- 中央必須有一個主符號：AI 核心、原型符號、動物抽象形或幾何圖騰。
- 外圍可以有細緻神秘邊框，但不能喧賓奪主。
- 小尺寸 184px 時，仍要像一枚清楚的徽章。
- 避免寫實羽毛、寫實毛髮、寫實眼睛；使用金屬線、光線、電路、星座節點來表現生命感。
- 避免過度日式傳統紋樣，精神參考家徽，但成品必須是 AI 科技神秘學。
- 避免複雜背景、星圖堆疊、過多節點與過多裝飾線；先做強識別，再做氣氛。

黃金比例 / Fibonacci 構圖原則：

- 主要比例使用 φ = 1.618，輔助比例使用 0.618 與 0.382。
- 圖像中心安全區 760px 可視為主構圖直徑，重要元素落在 760 / 1.618 ≈ 470px 與 470 / 1.618 ≈ 290px 的圓環節點上。
- 中央核心約佔主構圖寬度的 38.2%，外圍家徽環約佔 61.8%。
- 動物圖騰的翼展、角度或流線不要任意拉滿，優先落在 0.618 的視覺半徑內。
- 可使用 Fibonacci spiral 的方向感暗示「演化、探索、擴張」，但不要畫成明顯教科書螺旋。
- 小尺寸顯示時，使用黃金比例控制留白：中心符號要最清楚，外圈細節退為氣氛。

動物表現原則：

- 動物感佔 30-40%，AI / 神秘幾何佔 60-70%，但最後必須收斂成一個簡潔符號。
- 不畫具體表情，不做卡通化，不做寵物感。
- 優先使用「動物靈」或「圖騰」：幾何輪廓、負空間、金箔線、少量電路紋。
- 每張圖保留塔羅牌式中心符號，動物融入符號外圍或背景能量。
- 小尺寸 184px 時，要先看見原型符號，其次才感覺到動物能量。
- 色彩跟隨原型專屬色，但可加入金色神秘線條與青綠 AI 光。

九大靈獸方向：

- AI探險家：隼，代表高空視野、未知邊界、第一個穿越迷霧。
- AI工匠：獾，代表耐性、打磨、挖掘與工具感。
- AI守護者：鯨，代表深層守護、情緒共鳴、龐大而溫柔的屏障。
- AI領航員：海豚，代表導航、群體智慧、穿越資料海流。
- AI策士：狐，代表局勢判讀、隱藏規則、安靜的機敏。
- AI發明家：蜂鳥，代表高速試作、微小但強烈的創造能量。
- AI交易員：獵豹，代表速度、時機、精準出手。
- AI導師：貓頭鷹，代表洞察、知識傳遞、夜間視野。
- AI建造者：象，代表結構、記憶、承重與可留下來的系統。

## 1.2 UI 插槽與安全區

圖片不是獨立海報，必須剛好放進現有 Alpha UI 的「蘿蔔坑」。

### 原型角色圖

- 前端路徑：`assets/archetypes/{key}.png`
- 建議來源尺寸：1024 x 1024 px
- 前端主要使用位置：結果頁身份揭曉徽章、未來原型卡片
- 顯示方式：`background-size: cover`
- 安全區：中心 760 x 760 px 必須完整可讀
- 外圈 12% 不放關鍵符號、臉、文字或重要細節
- 圖像要能在 168-184 px 的手機 UI 插槽中仍然一眼看懂
- 不建議把塔羅邊框做得太靠外，因為小尺寸會被裁切或看不清

### 測驗場景圖

- 前端路徑：`assets/scenes/scene-*.png`
- 建議來源尺寸：1200 x 720 px
- 前端主要使用位置：測驗頁兩張情緒選項卡片中的場景區
- 顯示方式：`background-size: cover`
- 安全區：中心 70% 保留主要圖像
- 左右邊緣可放氣氛元素，不放主體
- 缺圖時前端會顯示漸層與原型符號，不會破圖

### 分享卡圖

- 前端路徑：`assets/share-cards/{key}.png`
- 建議來源尺寸：1080 x 1440 px
- 安全區：上下各 160 px 不放重要主體
- 文字由前端或後製疊加，生成圖內不放可讀文字

## 7. 未來 MCP 圖片工廠接入方式

本輪不接 MCP，也不呼叫圖片生成 API。未來接入時建議流程如下：

1. 讀取 `data/archetypes.js` 與本文件中的 prompt。
2. 逐張生成原型角色圖與場景圖。
3. 產物寫入 `assets/archetypes/`、`assets/scenes/`、`assets/share-cards/`。
4. 以本機頁面檢查 390px mobile crop 是否清楚。
5. 不滿意的圖只重生單張，不改動前端流程。
6. 最後再考慮把生成任務包成可重跑的圖片工廠腳本。
